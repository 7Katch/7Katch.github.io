Modulo 01

Sistemi Operativi, Processi e Thread

Sistemi Operativi

L'intermediario fondamentale tra l'hardware del computer e i programmi applicativi. Gestisce l'allocazione delle risorse (CPU, memoria, periferiche), astrae la complessità dei componenti fisici e isola le esecuzioni per garantire stabilità e consistenza.

Processi

Un'istanza di un programma in esecuzione, dotata del proprio spazio di indirizzamento isolato (Codice, Dati, Heap, Stack). Ogni processo possiede un PCB (Process Control Block) che ne traccia lo stato, i registri della CPU e le risorse allocate dal sistema.

Thread

L'unità più piccola di utilizzo della CPU (spesso definiti processi leggeri). Più thread all'interno dello stesso processo condividono il segmento codice, dati e le risorse di sistema, ma mantengono uno stack privato e un program counter indipendente per l'esecuzione parallela o concorrente.

Modulo 02

CPU Scheduling, Sincronizzazione e Deadlocks

CPU Scheduling

L'algoritmo con cui lo scheduler decide quale thread pronto debba occupare la CPU. Nei sistemi embedded e real-time (RTOS) la pianificazione è deterministica e basata su priorità rigide, mentre nei sistemi operativi tradizionali adotta schemi a turni (Round-Robin) o code multilivello con feedback.

Sincronizzazione

I meccanismi di coordinamento tra processi/thread per evitare corruzioni di dati (Race Conditions) durante l'accesso a risorse condivise. Include costrutti ad esclusione mutua vincolata a un proprietario (Mutex) e contatori di permessi generici per l'allocazione simultanea (Semafori).

Deadlocks (Stallo)

Una condizione di blocco permanente in cui due o più thread si attendono a vicenda, bloccati su risorse incrociate. Si verifica al manifestarsi simultaneo delle quattro condizioni di Coffman: mutua esclusione, possesso e attesa, assenza di preemption, e attesa circolare.

Modulo 03

Main Memory e Virtual Memory

Main Memory (Memoria Principale)

Lo spazio hardware fisico (la RAM) in cui risiedono i dati e i codici pronti per la CPU. È mappata in indirizzi fisici sequenziali e richiede tecniche di allocazione controllata (paginazione o segmentazione) per ospitare in modo sicuro i segmenti dei processi.

Virtual Memory (Memoria Virtuale)

L'astrazione creata dal Sistema Operativo che separa la memoria logica del processo dalla memoria fisica reale. Ogni processo crede di avere a disposizione una mappa di indirizzi logici continua a partire da zero. La MMU traduce al volo questi indirizzi intermedi salvaguardando l'isolamento dei dati.

Modulo 04

I/O Systems, Mass-Storage e File System

I/O Systems (Sistemi di Input/Output)

Il sottosistema incaricato di gestire la comunicazione con i dispositivi esterni (mouse, tastiere, schede di rete). Sfrutta driver specifici, controller hardware e tecniche di interruzione o Direct Memory Access (DMA) per sollevare la CPU dal trasferimento dei singoli byte.

Mass-Storage Systems

I sistemi fisici di memorizzazione non volatile a lungo termine (Hard Disk, SSD, memorie Flash). Il kernel implementa algoritmi di ottimizzazione degli accessi per velocizzare la lettura/scrittura di blocchi binari grezzi sulla superficie fisica dei dischi.

File System

L'astrazione logica che organizza i blocchi grezzi del mass-storage in entità comprensibili all'utente (file e cartelle). Gestisce i metadati (permessi, estensioni, date di creazione) e traccia lo spazio libero/occupato sul disco tramite tabelle di allocazione (es. FAT, NTFS, ext4).

Modulo 05

Embedded Systems e Sicurezza

Embedded Systems (Sistemi Embedded)

Sistemi di calcolo dedicati a una singola funzione specifica, vincolati da risorse hardware minime (poca RAM, poca CPU, consumi energetici ridotti). Spesso utilizzano OS minimalisti o RTOS (come RIOT OS) dove il controllo dei thread e dello stack deve essere strutturato meticolosamente dal programmatore.

Sicurezza dei Sistemi

Insieme di tecniche volte a mitigare le vulnerabilità di memoria causate da una cattiva gestione del codice (come i Buffer Overflow). Si implementano contromisure a runtime come l'ASLR (casualizzazione dello spazio di indirizzamento) o controlli automatici preventivi sui puntatori delegati al compilatore.

✅ Nota di studio: L'isolamento della Memoria Virtuale e la sincronizzazione tramite Mutex costituiscono le fondamenta hardware e software necessarie a rendere sicura la concorrenza in un sistema multithread.
