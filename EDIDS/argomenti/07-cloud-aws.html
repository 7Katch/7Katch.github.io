Il Paradigma Cloud

Il Cloud Computing è l'erogazione di risorse informatiche (server, storage, database, rete) tramite Internet ("il cloud") con tariffazione a consumo.

La metafora "Pets vs Cattle"

Nel modello on-premise tradizionale, i server erano come "animali domestici" (Pets): macchine costose, curate manualmente, con nomi propri, e se si ammalavano venivano riparate. Nel cloud moderno, i server sono considerati "bestiame" (Cattle): risorse numerate, usa-e-getta, che vengono distrutte e ricreate automaticamente quando sono difettose o quando il carico aumenta (scalabilità orizzontale).

Modelli di Servizio
  • IaaS (Infrastructure as a Service): Noleggi l'hardware base (server virtuali, reti, storage). Massima flessibilità ma massima responsabilità di gestione (es. aggiornare il sistema operativo). Esempio: AWS EC2.
  • PaaS (Platform as a Service): Fornisce un ambiente di sviluppo e deploy già pronto. Ti preoccupi solo di scrivere il codice e i dati; il provider gestisce l'hardware e il sistema operativo. Esempio: Heroku, AWS Elastic Beanstalk.
  • SaaS (Software as a Service): Applicazioni web finite pronte all'uso, gestite interamente dal fornitore. Esempio: Google Workspace, Office 365, Dropbox.

Servizi Core AWS

Amazon Web Services (AWS) offre centinaia di servizi, ma le fondamenta su cui poggiano tutte le architetture moderne si dividono in tre categorie principali: Compute, Storage e Database.

Compute (Calcolo)
  • Amazon EC2: Macchine virtuali (IaaS) configurabili per CPU e RAM. Ti danno pieno controllo sul sistema operativo.
  • Amazon EKS / ECS: Servizi specializzati per orchestrare flotte di container Docker.
  • AWS Lambda: Paradigma Serverless. Non noleggi nessun server: scrivi solo una funzione di codice e AWS la esegue solo quando viene scatenato un evento. Paghi al millisecondo di esecuzione.
Storage e Database
  • S3 (Simple Storage Service): Object Storage infinito ed economico. Perfetto per file, immagini e backup. I file sono accessibili tramite URL.
  • EBS (Elastic Block Store): Dischi virtuali da "attaccare" alle macchine EC2. Fungono da dischi rigidi di sistema.
  • RDS (Relational Database Service): Database relazionali gestiti (MySQL, PostgreSQL). AWS si occupa di backup, patch e alta affidabilità.
  • DynamoDB: Database NoSQL proprietario di AWS, progettato per scalare orizzontalmente all'infinito mantenendo latenze di un singolo millisecondo.

Sicurezza e Rete (VPC)

Nessuna infrastruttura Cloud è sicura senza un isolamento di rete adeguato.

Amazon VPC (Virtual Private Cloud)

Una porzione logicamente isolata del cloud AWS dedicata unicamente a te. È il tuo data center virtuale.

  • Subnet Pubbliche: Porzioni di rete che hanno una via di fuga verso Internet (tramite un Internet Gateway). Vi si mettono, ad esempio, i server Web (Frontend).
  • Subnet Private: Porzioni di rete isolate dal mondo esterno. I server qui dentro non hanno IP pubblico. Vi si mettono Database e server Backend sensibili, accessibili solo dalle subnet pubbliche interne.
  • Security Groups: Firewall virtuali applicati direttamente al server (es. "Permetti traffico solo sulla porta 80 e 443"). Sono Stateful.
  • Network ACL: Regole di firewall applicate all'intera subnet. Sono Stateless.

Infrastructure as Code (IaC)

Nel cloud, configurare server cliccando sui bottoni del pannello web (AWS Console) porta inevitabilmente a errori umani e impedisce di replicare l'infrastruttura rapidamente.

Terraform & AWS CDK

L'IaC è la pratica di gestire le risorse informatiche attraverso file di testo dichiarativi leggibili dall'uomo.

  • Invece di creare un database a mano, si scrive nel codice: "Voglio un database PostgreSQL versione 13 con 20GB di spazio".
  • Strumenti come Terraform leggono questo codice e chiamano automaticamente le API di AWS per costruire esattamente l'architettura richiesta in pochi secondi.
  • Essendo puro testo, l'infrastruttura entra nel controllo di versione (Git), permettendo di tracciare ogni modifica strutturale (chi ha aperto quella porta firewall e quando?) e di distruggere/ricreare interi ambienti di test (Staging/Production) con un solo comando.
